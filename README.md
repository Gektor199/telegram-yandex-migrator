# Migrator

Многопользовательский сервис для переноса экспортов Telegram Desktop в Яндекс Мессенджер.

## Что внутри

- `FastAPI` веб-приложение
- `PostgreSQL` для пользователей, задач, событий и runtime-настроек
- отдельный `worker`, который забирает задания из очереди и выполняет импорт
- OIDC SSO для входа пользователей
- админ-панель для runtime-настроек и SSO-реквизитов
- запуск всей системы через Docker Compose

## Поток работы

1. Пользователь входит через SSO.
2. Загружает ZIP-архив Telegram Desktop export с `result.json`.
3. Задание сохраняется в БД со статусом `queued`.
4. Worker забирает задание и переносит сообщения в Яндекс Мессенджер.
5. Пользователь и админ видят прогресс, ошибки и историю задач.

## Быстрый старт в Docker

```bash
cd /path/to/migrator
docker compose up --build
```

После старта:

- веб: [http://127.0.0.1:8080](http://127.0.0.1:8080)
- callback для SSO: `http://127.0.0.1:8080/auth/callback`

## Установка на новый сервер

Для развёртывания как install bundle используйте:

- `INSTALL.md`
- `installer/build-installer.sh` — сборка архива для передачи на другой сервер
- `install.sh` — bootstrap-установщик на целевой машине

Сценарий:

1. собрать архив установщика;
2. передать его на сервер;
3. распаковать;
4. выполнить `bash install.sh`;
5. заполнить временную локальную setup-страницу;
6. дождаться запуска `docker compose`.

## Важные переменные

- `DATABASE_URL` — строка подключения к Postgres
- `SECRET_KEY` — ключ для подписи сессий
- `MAINTENANCE_MODE` — включает сервисную страницу для всех маршрутов, кроме `/health` и `/static`
- `MAX_ARCHIVE_SIZE_MB` — максимальный размер ZIP-архива на уровне приложения
- `PUBLIC_BASE_URL` — внешний URL сервиса
- `YANDEX_BOT_TOKEN` — bootstrap токен Яндекс-бота из `.env` или окружения. В коде дефолтного токена нет.
- `MAX_PARALLEL_JOBS` — общий лимит параллельных импортов
- `MAX_PARALLEL_JOBS_PER_USER` — лимит параллельных импортов на одного пользователя
- `OIDC_SERVER_METADATA_URL` — `.well-known/openid-configuration`
- `OIDC_CLIENT_ID`
- `OIDC_CLIENT_SECRET`
- `ADMIN_EMAILS` — CSV email-админов
- `ADMIN_DOMAINS` — CSV доменов, которые автоматически получают админский доступ

## Runtime-настройки из админки

- токен бота Яндекс Мессенджера
- базовый URL Yandex Bot API
- лимит параллельных импортов worker'а
- лимит параллельных импортов на пользователя

## Локальная отладка без SSO

Если не задавать OIDC-параметры, страница `/login` переключится в локальный режим входа локальным администратором. Это предназначено только для разработки.
